# M. Omar Farooq — Digital CV Project Context

## What is this project?

A static portfolio website for **Omar Farooq**, a Senior Platform Architect with 20+ years experience building mission-critical digital infrastructure for the United Nations and international development organizations (IOM, UNICEF via IRC). The site showcases his work on six major platforms including the UN Migration Network Hub, CLIMB platform, Global Migration Data Portal, and others.

**Live at**: `https://ahaomar.github.io/`

## Technology Stack

- **Type**: Static HTML/CSS/JavaScript (no build tools)
- **HTML5**: Semantic markup with accessibility features
- **CSS3**: Custom properties in CSS file; responsive grid/flexbox layouts
- **JavaScript**: Vanilla ES6+ DOM manipulation only (~20 lines total)
- **Hosting**: GitHub Pages

**No frameworks, no bundlers, no transpilers.** Pure static files served directly.

## Design System (from style.css)

**Color palette:**
- `--bg`: #201E1A (charcoal primary background)
- `--bg-2`: #2A2721 (secondary surface)
- `--bg-3`: #171613 (darkest sections like CTA band)
- `--bone`: #EDE7DC (primary text color)
- `--bone-soft`: #C7C0B1 (secondary text, muted content)
- `--muted`: #948C7B (tertiary/metadata colors)
- `--copper`: #C08850 (accent/highlight primary)
- `--copper-soft`: #8C5A32 (darker copper for borders/strokes)
- `--line`: #3D392F (subtle dividers, grid lines)

**Typography:**
- **Serif**: 'Piazzolla' (headings - h1, h2; pull quotes)
- **Sans-serif**: 'Manrope', system fonts (-apple-system, Segoe UI) for body text
- **Monospace**: 'JetBrains Mono' (meta labels, tags, links, badges)

**Layout patterns:**
- Container width: `var(--wide)` = 1080px, centered with auto margins
- Constrained content column: `var(--measure)` = 700px for single-column sections
- Grid system: CSS grid and flexbox throughout; responsive breakpoints at 760px/640px/820px

**Visual motifs (consistency across pages):**
1. Isometric platform-stack SVG diagram in hero section
2. Connected timeline spine with circular markers for career history
3. Copper-colored rule dividers separating content blocks
4. Consistent link styling: copper by default, bone on hover
5. Same header/footer structure on all 4 main pages + blog post wrapper

## Conventions & Patterns to Follow

### HTML Structure
- All 4 main pages (`index`, `projects`, `blog`, `contact`) follow identical pattern:
  - `<header class="topbar">` with logo link and nav menu (sticky, dark background)
  - `<main>` content sections separated by `<hr class="rule">` or CTA bands
  - Content in `.wrap` container for max-width constraints
  - Single `<section>` per semantic area; use `.tight` subsections sparingly
- Blog posts nest inside blog.html with `article.post h1`, not separate full pages (yet)

### CSS Architecture
- Custom properties at root level; no nested selectors or inheritance hacks needed
- Utility-like approach: classes represent conceptual elements (`spine`, `.kicker`, `.tag`) rather than mixins
- Media queries use mobile-first implicitly but all content is responsive by default (no base styles for desktop)

### JavaScript Usage
- Single `main.js` file with DOMContentLoaded wrapper
- Only nav-toggle hamburger functionality currently; keep other scripts minimal or inline if needed

### Writing Style & Tone
Content reflects: enterprise-scale architecture, governance-focused thinking, humanitarian impact. Use measured language that conveys experience without hyperbole ("architected", "engineered" for systems I led). The portfolio is written in third person about Omar himself (e.g., "I am a Senior Drupal Developer...").
